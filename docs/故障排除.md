# Nova 故障排除

## 快速指令

```bash
# nova-server
nova status       # 查看 nova-server 狀態
nova start        # 啟動 nova-server
nova stop         # 停止 nova-server
nova restart      # 重啟 nova-server
nova logs         # 查看 nova-server 日誌
nova fix          # 一鍵修復（kill → clean → start → status）

# nova-llm
nova llm-status   # 查看本地 LLM 狀態
nova llm-start    # 啟動本地 LLM (launchd)
nova llm-stop     # 停止本地 LLM (launchd)
nova llm-restart  # 重啟本地 LLM (launchd)
nova logs-llm     # 查看 LLM 日誌

# 共用
nova ps           # 列出所有 nova 相關進程
nova kill         # 強制終止所有 nova 進程
nova clean        # 清除 lockfile + 快取
nova hooks        # 查看 hook-client 除錯日誌
nova help         # 顯示所有指令
```

## 常見問題

### Claude Code 卡住不動

**症狀**：輸入後無回應，游標不動。

**原因**：hook-client.js 在等待 nova-server 回應，但 nova-server 不可用且 auto-start 失敗。

**修復**：
```bash
# 另開 terminal
nova fix
```

**根因（已修復 2026-03-15）**：hook-client.js spawn nova-server 時缺少 `detached: true`，導致 hook-client 等待 nova-server 退出（永遠不會）。

### hook error 但操作仍成功

**症狀**：`PreToolUse:Write hook error` 出現在操作訊息中，但文件仍然寫入成功。

**原因**：settings.json 中的 hook 命令參數含有 `|`（如 `Write|Edit`），被 shell 解析為管道符。

**影響**：hook 的 JSON 輸出被管道到不存在的命令，Claude Code 讀不到 block 決策 — guard 保護形同虛設。

**修復**：命令參數加引號。

```json
// ❌ 錯誤
"command": "bun ~/.claude/hooks/hook-client.js PreToolUse Write|Edit"

// ✅ 正確
"command": "bun ~/.claude/hooks/hook-client.js PreToolUse 'Write|Edit'"
```

**根因（已修復 2026-03-15）**：shell 把 `|` 當管道符，`Write` 部分正常執行但 stdout 被吞，`Edit` 作為獨立命令失敗（exit 127）。

### nova-server 自己 crash

**症狀**：`nova status` 顯示未運行，但之前是正常的。

**診斷**：
```bash
nova logs     # 查看 crash 前的日誌
nova start    # 重新啟動
```

### 本地 LLM 無回應

**症狀**：`nova llm-status` 顯示未運行，或 maintainer.js / learner.js timeout。

**診斷**：
```bash
nova llm-status   # 檢查狀態
nova logs-llm     # 查看日誌（模型載入失敗、OOM 等）
nova llm-restart  # 重啟（模型載入需 5-10 秒）
```

**常見原因**：
- 記憶體不足（Qwen3.5-35B 需要 ~19GB RAM）
- pyenv Python 版本升級後 binary copy 過期 → `cp $(pyenv which python3.12) ~/.claude/bin/nova-llm`

### 背景進程堆積

**症狀**：`nova ps` 顯示多個 maintainer.js 或 learner.js 進程。

**原因**：本地模型離線時，`askLocalModel()` 等待 300 秒 timeout。每次 session 結束都會觸發新進程。

**修復**：
```bash
nova kill     # 終止所有相關進程
```

### lockfile 殘留

**症狀**：`nova status` 顯示 `⚠️ lockfile 殘留`。

**原因**：hook-client 在 auto-start 過程中異常退出，lockfile 未清除。後續 auto-start 以為有人正在啟動，等 1 秒後放棄。

**修復**：
```bash
nova clean    # 清除 lockfile + statusline 快取
```

### Statusline 顯示 nova ✗ 但實際正常

**原因**：statusline 有 3 秒快取（`/tmp/nova-status-cache`）。nova-server 剛啟動時快取可能還是舊的。

**修復**：等 3 秒自動更新，或 `nova clean` 手動清快取。

### 活動監視器找不到進程

**原因**：macOS 活動監視器顯示 kernel `p_comm`（binary basename），不是 `argv[0]`。

**解決方式**：所有 nova 進程使用 **binary copy** 命名（非 symlink、非 `exec -a`）。搜尋 `nova` 即可找到。

**runtime 升級後需重新複製**：
```bash
cp $(which bun) ~/.claude/bin/nova-server
cp $(pyenv which python3.12) ~/.claude/bin/nova-llm
```

## 架構圖

```
macOS 登入
  └─ launchd
       └─ com.overtone.vllm-mlx.plist
            └─ nova-llm (port 8000) ─── 本地 LLM 推論

Claude Code
  │
  ├─ hook 觸發 ──→ hook-client.js ──fetch──→ nova-server:3457/dispatch
  │                    │                         │
  │                    │ (server 沒跑)            ├─ guards.js (Write/Edit/Bash 保護)
  │                    ├─ autoStart()             ├─ metrics.js (統計)
  │                    │   └─ spawn nova-server   ├─ notification.js (macOS 通知)
  │                    │      (detached+unref)    └─ ...其他 handler modules
  │                    │
  │                    └─ fallback import
  │                       (server 仍不可用)
  │
  ├─ session 結束 ──→ maintainer.js (背景) ──→ nova-llm:8000
  └─ session 結束 ──→ learner.js (背景) ────→ nova-llm:8000
```

## 進程命名

所有 nova 進程使用 binary copy 確保在 `ps`、`pgrep`、macOS 活動監視器中可辨識。

| 進程 | binary 路徑 | runtime 來源 | 用途 |
|------|-------------|-------------|------|
| nova-server | `~/.claude/bin/nova-server` | `bun` | hook dispatch (port 3457) |
| nova-llm | `~/.claude/bin/nova-llm` | `python3.12` | 本地 LLM (port 8000) |

## 日誌位置

| 日誌 | 路徑 | 內容 |
|------|------|------|
| nova-server | `/tmp/nova-server.log` | server 啟動、crash、handler 載入 |
| nova-llm | `/tmp/vllm-mlx.log` | LLM 啟動、模型載入、推論錯誤 |
| hook-client | `/tmp/hook-client-debug.log` | 每次 hook 的 stdin/dispatch 耗時、決策結果 |
| statusline 快取 | `/tmp/nova-status-cache` | UP 或 DOWN（3 秒更新） |
| lockfile | `/tmp/nova-server.lock` | auto-start 防重複啟動 |
