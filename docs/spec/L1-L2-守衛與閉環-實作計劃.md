## 閉環路徑

### 迴路 1：學習閉環（instinct loop）

```
[寫入方]                    [分析方]                    [消費方]
7 個 flow hooks ──────> on-end-flow.js ──────> on-start-flow.js
    │                       │                       │
    │ writeFlowEvent()      │ session-analyzer.js   │ readInstincts()
    │ → JSONL               │ → extractObservations │ → additionalContext
    │                       │ → appendInstinct()    │   注入 session context
    v                       v                       v
 events.jsonl          instincts.jsonl         Main Agent prompt
```

**資料格式**（instincts.jsonl 每行）：
```json
{
  "ts": "2026-03-14T...",
  "sid": 42,
  "type": "observation",
  "category": "guard_pattern|error_pattern|efficiency",
  "text": "本 session 中 pre-bash-guard 觸發 3 次，主要攔截 rm -rf 命令",
  "confidence": 0.8,
  "ttl": 7
}
```

### 迴路 2：守衛反饋（guard feedback loop）

```
[觸發方]                    [記錄方]                    [趨勢方]
pre-bash-guard.js ──────> writeFlowEvent() ──────> session-analyzer.js
cost-guard.js                                          │
recursion-guard.js                                     │ guard 觸發頻率分析
                                                       v
                                                  instinct: "cost 連續 3 session 超標，建議降級 model"
```

### 迴路 3：Debug 消費（人工迴路）

```
[寫入方]                    [查詢方]                    [消費方]
所有 hooks ──────────> flow-cli.js ──────────> 開發者 terminal
    │                      │
    │ writeFlowEvent()     │ queryEvents --type guard_block --last 1h
    │ → JSONL              │ detectAnomalies → 連續失敗偵測
    v                      v
 events.jsonl          格式化輸出 + 異常警告
```

## 步驟（6 步，每步 ≤ 3 檔案變更）

### Phase 1：守衛強化（可並行）

**步驟 1：修復 pre-edit-guard + 新增 cost-guard**
- 執行者：executor (sonnet)
- 檔案：
  1. `hooks/scripts/tool/pre-edit-guard.js` -- 移除 manage-component.js 引用，改為 advisory block（reason 改為「直接修改 ~/.claude/ 元件，確認你知道影響範圍」）
  2. `hooks/scripts/tool/cost-guard.js` -- 新增成本守衛
  3. 測試檔案 `tests/guards/cost-guard.test.js`
- **cost-guard.js 規格**：
  - 掛載：PreToolUse / Bash（與 pre-bash-guard 同事件，但不同 hooks 陣列項）
  - 行為：讀取 `$TMPDIR/overtone-cost-tracker.json`，累計 session token 估算
  - 閾值：warn @ 500K tokens（additionalContext 提醒），block @ 2M tokens
  - evaluate() 純函式，不碰 I/O
  - 估算方式：每次 Bash call +500 tokens，每次 Agent call +5000 tokens（粗估，後續可調）

**步驟 2：新增 recursion-guard**
- 執行者：executor (sonnet)
- 檔案：
  1. `hooks/scripts/tool/recursion-guard.js` -- 新增遞迴防護
  2. 測試檔案 `tests/guards/recursion-guard.test.js`
- **recursion-guard.js 規格**：
  - 掛載：PreToolUse / Agent（與 pre-agent-flow 同事件同 matcher）
  - 行為：讀取 `$TMPDIR/overtone-agent-depth.json`，追蹤 agent 委派深度
  - 深度計算：每次 Agent call depth++，SubagentStop 時 depth--（透過共享檔案）
  - 閾值：block @ depth > 4（Main → D1 → D2 → D3 → D4 = 4 層已是上限）
  - evaluate() 純函式

### Phase 2：閉環基礎設施（sequential -- instinct-store 先於 analyzer）

**步驟 3：建立 instinct 持久化層**
- 執行者：executor (sonnet)
- 檔案：
  1. `scripts/lib/instinct-store.js` -- JSONL 持久化
  2. 測試檔案 `tests/lib/instinct-store.test.js`
- **instinct-store.js 規格**：
  - 檔案位置：`$TMPDIR/overtone-instincts.jsonl`
  - `appendInstinct({category, text, confidence, ttl})` -- 寫入一行 JSON，自動加 ts/sid
  - `readInstincts(limit=10)` -- 讀最近 N 條未過期的 instinct
  - `pruneInstincts(maxAge=7)` -- 刪除超過 N 天的 instinct
  - TTL 機制：每條 instinct 有 ttl（天），readInstincts 自動過濾過期項目
  - 格式與 event-writer.js 一致（JSONL + appendFileSync）

**步驟 4：建立 session 分析器 + 修改 on-end-flow**
- 執行者：executor (sonnet)
- 檔案：
  1. `scripts/lib/session-analyzer.js` -- 純函式分析
  2. `hooks/scripts/session/on-end-flow.js` -- 修改：session 結束時呼叫分析
  3. 測試檔案 `tests/lib/session-analyzer.test.js`
- **session-analyzer.js 規格**：
  - `analyzeSession(events[])` -- 接收一個 session 的所有 flow events
    - 計算：guard 觸發次數、tool 使用分布、agent 委派鏈、session 時長、錯誤數
    - 回傳：`{guardBlocks, toolDistribution, agentChain, duration, errorCount}`
  - `extractObservations(analysis)` -- 從分析結果產生 instinct observations
    - 規則：guardBlocks > 2 → 產生 guard_pattern observation
    - 規則：errorCount > 3 → 產生 error_pattern observation
    - 規則：duration > 30min 且 tool 使用 < 10 → 產生 efficiency observation（卡住徵兆）
    - 回傳：`[{category, text, confidence, ttl}]`
- **on-end-flow.js 修改**：
  - 原行為保留（writeFlowEvent session_end）
  - 新增：`getSessionEvents(currentSid)` → `analyzeSession()` → `extractObservations()` → 逐一 `appendInstinct()`

### Phase 3：閉環注入（依賴 Phase 2）

**步驟 5：修改 on-start-flow + settings.json**
- 執行者：executor (sonnet)
- 檔案：
  1. `hooks/scripts/session/on-start-flow.js` -- 修改：注入 instinct context
  2. `settings.json` -- 新增 cost-guard + recursion-guard 掛載
  3. 測試檔案 `tests/hooks/on-start-flow.test.js`
- **on-start-flow.js 修改**：
  - 原行為保留（writeFlowEvent session_start）
  - 新增：`readInstincts(5)` → 格式化為 additionalContext 字串
  - additionalContext 格式：
    ```
    ## 歷史觀察（自動產生，僅供參考）
    - [guard_pattern] 最近 session 頻繁觸發 bash guard，注意命令安全性
    - [efficiency] 上次 session 花 45 分鐘但只執行 8 個 tool call，可能有卡住情況
    ```
  - Hook 輸出：`{ additionalContext: "..." }`（UserPromptSubmit hook 的 hookSpecificOutput）
  - **注意**：SessionStart hook 的 hookSpecificOutput 只支持 `additionalContext`

### Phase 4：Debug 工具（與 Phase 2-3 獨立，可並行）

**步驟 6：建立 flow-cli + anomaly-detector**
- 執行者：executor (sonnet)
- 檔案：
  1. `scripts/lib/anomaly-detector.js` -- 異常偵測規則
  2. `scripts/flow/flow-cli.js` -- CLI 入口
  3. 測試檔案 `tests/lib/anomaly-detector.test.js`
- **anomaly-detector.js 規格**：
  - `detectAnomalies(events[])` -- 掃描事件序列，回傳異常清單
  - ANOMALY_RULES：
    - `consecutive_failures`：連續 3+ 個 guard block → 警告
    - `long_session`：session 時長 > 60 分鐘且無 session_end → 警告
    - `deep_recursion`：agent_dispatch 深度 > 3 → 警告
    - `silent_session`：session 有 start 但 0 個 tool_use → 警告
  - 回傳：`[{rule, severity, description, events}]`
- **flow-cli.js 規格**：
  - 用法：`bun ~/.claude/scripts/flow/flow-cli.js [command] [options]`
  - 子命令：
    - `tail [--limit N]` -- 顯示最近 N 個事件（預設 20）
    - `sessions` -- 列出所有 session 索引
    - `session <sid>` -- 顯示特定 session 的事件
    - `guards [--last Nh]` -- 篩選 guard 相關事件
    - `anomalies` -- 執行異常偵測並輸出
  - 輸出格式：彩色 terminal 輸出（用 ANSI escape codes）
  - 每個子命令對應一個純函式（可測試）

## 不做什麼（明確排除）

1. **不建 heartbeat daemon**：L2 心跳系統需要獨立的 daemon 生命週期管理，與守衛/閉環是不同的子系統
2. **不建 queue guardian**：佇列守衛需要先有佇列系統，目前沒有 queue.js
3. **不建 session-spawner 遞迴防護**：session-spawner 不存在，遞迴防護在 agent 層級（recursion-guard）已覆蓋
4. **不修改 7 個 flow observer hooks 的核心邏輯**：只修改 on-end-flow 和 on-start-flow 的「新增行為」
5. **不建評分系統**：score-engine 是 L3 範疇（spec 中已有 overtone-子系統.md 描述），此次只建分析器
6. **不建 PreCompact hook**：context 壓縮前的狀態保存是獨立功能，不在本次範圍
7. **不建 Flow Visualizer UI 改動**：CLI 工具足以滿足 debug 需求，UI 已存在且獨立
8. **不建 PostToolUseFailure hook**：失敗後處理需要更多設計（retry 策略），留待後續
