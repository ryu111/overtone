# 競品基準（Competitor Benchmark）

> 基準日期：2026-03
> 資料來源：公開資訊 + 實際使用體驗
> 建議更新頻率：每季度人工審查一次

## 用途

供 retrospective agent 在六維度評估時對照參考。
不是用來貶低競品，而是建立客觀的相對定位感。

---

## 競品對照矩陣

| 維度 | Cursor | Windsurf | Devin | Claude Code + Overtone |
|------|--------|----------|-------|------------------------|
| **理解力** | 單檔案精準，跨檔案中等（依賴 codebase index） | 類似 Cursor，跨檔案稍弱 | 專案級理解強，可自主探索 repo | 專案級 + 領域理解（PM 訪談 → BDD → 實作閉環） |
| **創造力** | 依靠使用者引導，補全為主 | 依靠使用者引導，補全為主 | 有自主方案設計，但常保守 | 多方案比較 + RICE 評分 + architect agent 設計 |
| **美感** | 無 UI 生成（純程式碼工具） | 無 UI 生成 | 基本 UI，排版一般 | 內嵌 CSS 可用級，有設計感意識 |
| **細心** | 高（inline suggestions 精準） | 高（類似 Cursor） | 中（可能忽略邊界條件） | 中-高（BDD 驅動，tester agent 覆蓋邊界） |
| **完整度** | 低（片段補全，需手動整合） | 低（類似 Cursor） | 高（端到端執行，有自主除錯） | 高（workflow pipeline，DEV→TEST→REVIEW→RETRO→DOCS） |
| **架構能力** | 低（片段建議，無架構視野） | 低（類似 Cursor） | 中（有架構設計，但單一 agent） | 高（architect agent 專職設計，SOLID + BDD 驅動） |

## 各維度評分基準

### 1–5 量化說明

| 分數 | 說明 |
|------|------|
| 1 | 明顯不足，使用者需大量補救 |
| 2 | 有基本概念但有明顯缺口 |
| 3 | 達到基本可用，業界平均水準 |
| 4 | 超過平均，有明顯優勢 |
| 5 | 領先業界，接近最佳實踐 |

### 各競品參考總分（六維度，滿分 30）

| 工具 | 理解力 | 創造力 | 美感 | 細心 | 完整度 | 架構能力 | **總分** |
|------|--------|--------|------|------|--------|----------|---------|
| Cursor | 3 | 2 | 1 | 4 | 2 | 2 | **~14** |
| Windsurf | 3 | 2 | 1 | 4 | 2 | 2 | **~14** |
| Devin | 4 | 3 | 2 | 3 | 4 | 3 | **~19** |
| Claude Code + Overtone | 4 | 4 | 3 | 4 | 4 | 4 | **~23** |

> **注意**：這些是 2026-03 的主觀估計值，反映公開資訊和實際測試。不同任務類型下排名可能不同。

## 各競品特性說明

### Cursor
- **強項**：IDE 整合深，inline suggestion 精準，開發者體驗好
- **弱項**：無 workflow 概念，不能自主規劃跨模組任務
- **典型場景**：單一檔案編輯、函式補全、快速 refactor

### Windsurf
- **強項**：UI/UX 比 Cursor 友善，Cascade 功能可跨多檔案
- **弱項**：與 Cursor 類似，缺乏 agent pipeline 概念
- **典型場景**：類似 Cursor，更強調流暢度

### Devin
- **強項**：完全自主（含環境設定、除錯、PR），端到端交付
- **弱項**：單一 agent 架構，複雜決策品質不穩定；缺乏領域知識積累
- **典型場景**：「幫我完成整個功能」的明確任務

### Claude Code + Overtone
- **強項**：多 agent 協作（planner/architect/developer/tester/reviewer），BDD 驅動，知識積累飛輪（experience-index），支援 PM 訪談 → 自動工作流
- **弱項**：需要 Overtone 設定，上手成本較高；UI 生成能力仍在成長中
- **典型場景**：複雜功能開發、跨領域系統設計、需要可維護性和測試覆蓋的任務

---

## 使用說明

1. **六維度評估時**：選取對應行，對照「Claude Code + Overtone」欄位
2. **競品對標**：每個維度先給出 X/5 得分，再與競品欄位對比
3. **總分換算**：六維度加總後對比表格底部的競品參考總分
4. **解讀**：分數只是相對定位，重點是識別各維度的具體優/弱項
