# 繁體中文技術寫作慣例

> 來源：教育部國語辭典 + 業界慣例 + Overtone 專案規範

## 一、中英混排規則

### 空格規則

| 規則 | 正確 | 錯誤 |
|------|------|------|
| 中文與英文之間加半形空格 | `執行 bun test 確認` | `執行bun test確認` |
| 中文與數字之間加半形空格 | `共 12 個項目` | `共12個項目` |
| 英文與半形標點之間不加空格 | `timeout（60 分鐘）` | `timeout （ 60 分鐘 ）` |
| 全形標點前後不加空格 | `完成，接著執行` | `完成 ， 接著執行` |
| 程式碼區塊（backtick）兩側加空格 | ``使用 `readFile` 讀取`` | ``使用`readFile`讀取`` |

### 標點符號規則

| 符號 | 中文語境（全形） | 程式碼/英文語境（半形） |
|------|------------------|------------------------|
| 逗號 | `，` | `,`（僅在程式碼內） |
| 句號 | `。` | `.`（僅在程式碼內） |
| 冒號 | `：` | `:`（僅在程式碼/鍵值對內） |
| 分號 | `；` | `;`（僅在程式碼內） |
| 括號 | `（）` | `()`（僅在程式碼/英文詞後） |
| 引號 | `「」`（外層）`『』`（內層） | `""`（僅在程式碼內） |
| 問號 | `？` | `?`（僅在程式碼內） |
| 驚嘆號 | `！` | `!`（僅在程式碼內） |
| 破折號 | `——`（兩個 em dash） | `--`（僅在 CLI 參數） |
| 刪節號 | `……`（兩個） | `...`（程式碼展開運算子） |

```javascript
// 正確：中文句子用全形標點
// 「此函式負責讀取佇列狀態，回傳下一個待執行項目。」

// 正確：程式碼註解中的技術描述用半形
// timeout: 60000 (ms)

// 錯誤：中文句子混用半形標點
// ❌ 「此函式負責讀取佇列狀態,回傳下一個待執行項目.」
```

## 二、技術術語保留規則

### 保留英文的術語（不翻譯）

| 類別 | 範例 |
|------|------|
| 程式語言關鍵字 | `function`、`const`、`async`、`await` |
| 框架/工具名稱 | Bun、Node.js、Claude Code、htmx、Alpine.js |
| API 名稱 | `readFile`、`spawnSession`、`updateStateAtomic` |
| 協定/格式 | HTTP、JSON、JSONL、SSE、WebSocket |
| 開發術語 | commit、PR、merge、hook、plugin、agent |
| CLI 指令 | `bun test`、`claude -p`、`git status` |
| 設計模式 | singleton、strategy、observer、middleware |
| 錯誤類型 | `TypeError`、`ValidationError`、`ENOENT` |

### 翻譯為中文的術語

| 英文 | 繁體中文 | 說明 |
|------|----------|------|
| file | 檔案 | 不用「文件」（文件 = document） |
| document/documentation | 文件 | |
| directory/folder | 目錄 | 不用「資料夾」（技術文件偏好「目錄」） |
| function | 函式 | 不用「函數」（台灣慣用「函式」） |
| parameter | 參數 | |
| variable | 變數 | |
| array | 陣列 | 不用「數組」（大陸用法） |
| object | 物件 | 不用「對象」（大陸用法） |
| string | 字串 | 不用「字符串」（大陸用法） |
| module | 模組 | 不用「模塊」（大陸用法） |
| server | 伺服器 | |
| client | 客戶端 | |
| queue | 佇列 | 不用「隊列」（大陸用法） |
| thread | 執行緒 | 不用「線程」（大陸用法） |
| process | 程序 / 流程 | 視語境：OS process = 程序，workflow process = 流程 |
| callback | 回呼 | 或直接保留 callback |
| interface | 介面 | 不用「接口」（大陸用法） |
| memory | 記憶體 | 不用「內存」（大陸用法） |
| recursion | 遞迴 | 不用「遞歸」（大陸用法） |
| cache | 快取 | 不用「緩存」（大陸用法） |
| compile | 編譯 | |
| debug | 除錯 | 或直接保留 debug |

## 三、術語判斷決策樹

```
❓ 這個術語是程式碼的一部分嗎？（函式名、CLI 指令、API）
   → 是 → 保留英文原文，用 backtick 包裹：`spawnSession`
   → 否 ↓

❓ 這個術語是工具/框架的專有名稱嗎？
   → 是 → 保留英文原文，首字母大寫：Bun、Claude Code
   → 否 ↓

❓ 這個術語有常用的繁體中文對應嗎？
   → 是 → 使用繁體中文：「陣列」「佇列」「模組」
   → 否 ↓

❓ 這個術語在業界普遍保留英文？
   → 是 → 保留英文：commit、PR、hook、agent
   → 否 → 翻譯為繁體中文
```

## 六、數字與單位

| 規則 | 正確 | 錯誤 |
|------|------|------|
| 阿拉伯數字與中文之間加空格 | `共 3 個 agent` | `共3個agent` |
| 技術數值用阿拉伯數字 | `timeout 60 秒` | `timeout 六十秒` |
| 序數用阿拉伯數字 | `第 12 項偵測` | `第十二項偵測` |
| 百分比用半形 | `95%` | `９５％` |
| 檔案大小用英文單位 | `50 KB` | `50 千位元組` |
| 時間單位可用中文 | `60 分鐘` / `60 min` | |
