# 常駐服務 — 章文件

> 最後更新：2026-03-15

## 概述

Nova Server 是 nova 唯一常駐 daemon，將 9 個獨立 hook 腳本整合為單一 HTTP 服務（port 3457）。同時承載 hook dispatch、Flow Visualizer UI、SSE 廣播、JSONL 持久化、模組熱重載。

## 架構圖

```
Claude Code
    ↓ stdin JSON
hook-client.js (PreToolUse Bash)
    ↓ HTTP POST /dispatch
server.js (port 3457, 常駐)
    ├── handlerMap 路由
    │   ├── bash-guard.js    → PreToolUse:Bash
    │   ├── edit-guard.js    → PreToolUse:Write, PreToolUse:Edit
    │   ├── flow-observer.js → SessionStart, SessionEnd, UserPromptSubmit,
    │   │                      SubagentStop, PreToolUse:Agent, PostToolUse
    │   ├── notification.js  → Notification
    │   └── metrics.js       → 可觀測層（subscriber，非 handler）
    ├── 結果聚合（block AND 語意）
    ├── xstream event$ pipeline
    │   ├── writeFlowEvent() → JSONL 持久化
    │   ├── pool.broadcast() → SSE 推送
    │   └── metrics.onEvent() → 計數 + 異常偵測
    └── Flow Visualizer 路由（/, /events, /api/*)
         ↓
    Flow Visualizer UI (http://localhost:3457)
```

## 模組介面

### Handler 格式

```javascript
module.exports = {
  on: {
    'EventType[:Matcher]': (input) => ({
      decision: 'allow' | 'block',
      reason?: string,
      events?: FlowEvent[],
    }),
  },
};
```

### Nova Server API

| 路徑 | 方法 | 說明 |
|------|------|------|
| `POST /dispatch` | POST | `{ eventType, matcher?, input }` → `{ decision, reason? }` |
| `GET /health` | GET | `{ status, uptime, modules, connections, metrics, anomalies }` |
| `POST /modules/reload` | POST | 重新載入模組 → `{ status, modules }` |

### 路由優先順序

1. **精確匹配**：`EventType:Matcher`（如 `PreToolUse:Bash`）
2. **| 展開**：`Write|Edit` → 分別查找 `PreToolUse:Write` 和 `PreToolUse:Edit`
3. **寬鬆匹配**：只用 `EventType`（如 `PostToolUse`）
4. **去重**：同一函式參考只執行一次

### 結果聚合

- **block AND 語意**：任一 handler 返回 block → 整體 block
- **events 合併**：所有 handler 的 events 陣列合併後寫入 JSONL + SSE

## 檔案結構

| 檔案 | 位置 | 用途 |
|------|------|------|
| server.js | `~/.claude/hooks/` | 常駐 daemon（nova-server） |
| hook-client.js | `~/.claude/hooks/` | 統一 hook 入口（HTTP client + fallback） |
| bash-guard.js | `~/.claude/hooks/modules/` | Bash 安全守衛 |
| edit-guard.js | `~/.claude/hooks/modules/` | 編輯守衛 |
| flow-observer.js | `~/.claude/hooks/modules/` | Flow 事件觀察 |
| notification.js | `~/.claude/hooks/modules/` | macOS 通知 |
| metrics.js | `~/.claude/hooks/modules/` | 可觀測層（計數 + 異常偵測） |

## Fallback 策略

hook-client.js 的 `fetch` 失敗時（nova-server 未啟動），自動 import 舊 hook 腳本的 `evaluate()` 函式。舊腳本保留不刪。

## 功能演進

### Nova Server（2026-03-14）

- **摘要**：單一常駐 HTTP daemon 取代 9 個獨立 hook process
- **關鍵決策**：
  - Port 3457 合併（取代舊 flow-server）
  - 模組隔離：dispatch 同步回傳 decision，事件推送 fire-and-forget
  - xstream event$ 統一事件管道（取代 watchFile 500ms 輪詢）

### Hook Client 防呆強化（2026-03-15）

- **摘要**：修正 hook-client.js 的身份驗證、日誌導向、stdin 效能
- **關鍵決策**：
  - `/health` 雙重驗證：`status === 'ok'` 且 `title === 'nova-server'`
  - spawn 日誌導向 `/tmp/nova-server.log`（`openSync` fd，禁止 ignore）
  - `Bun.stdin.text()` 取代手動 stream（0.1ms vs 1.6ms）
  - fetch 加 `AbortSignal.timeout(3000)` 防卡住

### 可觀測層（2026-03-15）

- **spec**：`archive/2026-03-15_nova-server-observability/spec.md`
- **摘要**：event$ 新增 metrics subscriber，`/health` 擴充 metrics + anomalies
- **關鍵決策**：
  - 不動 dispatch 核心（同步 for loop），只在 event$ 加 subscriber
  - 純記憶體滑動窗口（5 分鐘），重啟歸零
  - 4 種異常偵測：high_dispatch_rate、consecutive_blocks、handler_error_spike、memory_pressure
