# 架構演進 — 章文件

> 最後更新：2026-03-15

## 概述

nova 從「16 角色 Pipeline」演進為「單腦 + D0-D4 深度路由 + Worker」。核心洞見：多 agent 有 3.6x token 懲罰、context 重建稅每次 ~15k tokens，SWE-bench 前 3 名全是單循環。

## 設計原則

| 代號 | 原則 | 說明 | 對應 rule |
|------|------|------|-----------|
| P1 | 預設單腦 | Main Agent 自己完成，只在有明確收益時才委派 Worker | `rules/深度路由.md` |
| P2 | 驗證靠執行 | 跑測試 > LLM review。測試通過 = 驗證完成 | `rules/失敗處理.md`（完成證據） |
| P3 | 知識與角色分離 | agent prompt 的最佳實踐 → Skill。角色人設 → 刪除 | `CLAUDE.md`（元件職責分離） |
| P4 | 並行靠工具 | 獨立子任務 → 背景 agent。多檔案 → 多 tool call | `rules/並行執行.md` |
| P5 | 成本意識 | 每個新增的 stage/agent 必須論證 token 成本 < 品質提升 | — |

## 深度路由

| 深度 | 流程 | 何時用 |
|------|------|--------|
| D0 | Main 直接做 | 一行修改、改設定 |
| D1 | Main 做 + 測試 | 小功能、bug fix、≤3 檔案 |
| D2 | planner → executor | 中型功能、跨模組 |
| D3 | planner → executor → reviewer | 安全敏感、需審查 |
| D4 | planner → 多 executor → reviewer | 大型、tasks.md 管控 |

完整定義：`~/.claude/rules/深度路由.md`

## Worker 定義

| Worker | 用途 | Model |
|--------|------|-------|
| planner | 產出 spec + design + tasks（注入 nova-spec skill） | opus |
| executor | 按 design 實作程式碼 | sonnet |
| reviewer | 獨立審查（安全敏感、跨模組） | opus |

與舊 Agent 的差異：prompt 從 2-4k tokens 縮減為 200-500 tokens，context 由 Main 直接傳遞（不重建）。

## 演進里程碑

### Pipeline → 深度路由（2026-03-12）

- **問題**：16 角色 Pipeline 導致 233k tokens 完成「加 flag 檔案」級任務（合理值 ~70k）
- **根因**：context 重建稅（~15k/次）+ 固定成本 stage（RETRO 35k、DOCS 56k）
- **決策**：刪除 Pipeline，Main 根據複雜度選深度，按需委派
- **佐證**：Scaling Agent Systems 論文（3.6x token 懲罰）、Anthropic 官方指南、SWE-bench 前 3 名

### Agent 18 → Worker 3（2026-03-12）

- **刪除**：architect、designer、tester、debugger、doc-updater、retrospective 等 15 個
- **知識去向**：agent prompt 的最佳實踐提取為 Skill（code-review、security-kb、testing 等）
- **保留**：developer → executor、code-reviewer → reviewer、planner（新增）
